Why are they useful?

Users can send data to your web application in three different ways.

Inside your controller action‚Äôs you can call params to access form & URL query data.

It‚Äôs a method that returns an ActionController::Parameters object, in practice it behaves a lot like a hash.

Understanding Strong Params

Next:

If you try to save an object to your database, but it doesn‚Äôt seem to work.

You may see this:

What does this mean?

Rails introduced the ‚Äústrong parameters‚Äù system, back in Rails 4 as a security feature.

It forces you to whitelist the attributes that can be saved.

This prevents an issue known as ‚Äúmass assignment‚Äù, which allows malicious users to set admin = true, or set other fields that normally they wouldn‚Äôt have access to.

You can whitelist fields like this:

  params.require(:book).permit(:title, :author, :cover, :language)
end

  2. params.require(:book).permit(:title, :author, :cover, :language)
  3. end

  params.require(:book).permit(:title, :author, :cover, :language)
end

How does this work?

  • The require method will look for params[:book] & raise an error if it‚Äôs not there.
  • The permit method is a list of allowed (but optional) attributes.

As a result, you get back a new params hash with these attributes, but now they‚Äôre clear to be saved to the database.

Example:

  1. Book.create(book_params)
